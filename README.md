# Калькулятор поступления микронутриентов на низкокалорийной диете

Веб-приложение на HTML, CSS и JavaScript для публикации без сервера Python для подбора типового суточного меню (1200–2000 ккал) и оценки поступления макро- и микронутриентов относительно суточных норм. Поддерживается моделирование дополнения рациона одним саше **OPTIFAST**.

## Возможности

- Выбор **пола**, **калорийности** (1200 / 1500 / 1800 / 2000 ккал) и **типа меню**:
  - домашнее;
  - офисное;
  - перекус в городе.
- Отображение меню по приёмам пищи (завтрак, обед, перекус, ужин).
- Панель макро- и микронутриентов с цветовой индикацией:
  - зелёный — ≥ 90% от нормы;
  - жёлтый — 60–89%;
  - красный — &lt; 60%.
- Переключатель **«Диета / Диета + 1 саше OPTIFAST»** — пересчёт поступления нутриентов и индикаторов с учётом состава одного саше.

## Локальный просмотр

Из папки `github-pages` запустите простой HTTP-сервер (нужен для загрузки JSON):

```bash
# Python
python -m http.server 8080

# или Node.js
npx serve .
```

Откройте в браузере: http://localhost:8080

## Публикация на GitHub Pages

### Вариант 1 — папка `/docs` или `/github-pages` в репозитории

1. Загрузите содержимое папки `github-pages` в репозиторий GitHub.
2. В настройках репозитория: **Settings → Pages**.
3. В **Source** выберите ветку `main` и папку с файлами (`/github-pages` или `/docs`).
4. Сохраните — сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.

### Вариант 2 — отдельный репозиторий `username.github.io`

Скопируйте содержимое `github-pages` в корень такого репозитория — сайт откроется по адресу `https://<username>.github.io/`.


Значения поступления нутриентов — **демонстрационные усреднённые оценки** по типовым блюдам. Приложение не предназначено для медицинской диагностики.

Нормы потребления основаны на методических рекомендациях **MP 2.3.1.0253-21** «Нормы физиологических потребностей в энергии и пищевых веществах для различных групп населения Российской Федерации» (утв. Роспотребнадзором 22.07.2021).

## Обновление данных

При изменении меню или состава OPTIFAST обновите файлы в `data/` и перезалейте на GitHub.
